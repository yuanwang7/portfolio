Part B - 2

Part B Challenges

One of the significant challenges is that the dataset does not contain latitude and longitude of the sports organisation. However, the google map API requires longitude and latitude to locate the organisation on the map.

For the sports activity visualisation, there is a lot of technology for me to learn. For example, How to connect the chart with the database,how to dynmacially change the chart when he data in the backend has been changed? How to build the database? How to obtain some meaning data for the useful such as total duration of activity. How to display data interactivly? In addition, As a team leader, I have to make sure each team member has accomplished equal amount of work and implement the MVP so that the website has basic functionality before partB demonstration.

Part B Process

Overall, I have designed and developed the visualisation of the user sports activity. For example, the total time of physical activty which is shown on the bar chart. Also, I designed a BMI chart for a user to check out their BMI changes over a year. In addition, the doughnut chart shows different sports user played over the years. First, I did some research about the competitive APIs of the chart visualisation. I tried the Google chart first. However, I found that Chart.js is more responsive. Therefore. I switched to Chart.js API. Since one of the challenges is the update the chart dynamically, therefore, I have to build a database with corresponding attributes and records. To obtain the total duration, I used aggregation SQL statement such as SUM. To make a chart changed automatically, I used AJAX. First, I wrote a PHP file which aggregates data from Mysql. Then, using Ajax to call the PHP file. Then I started implementing the map interaction. One of the features is to locate the club on the map. However, there is no longitude and latitude information, which I can not mark them on the map directly. Thanks for the tutor's help, I tried Google geocode, which can get the longitude and latitude. The sports filter implementation is not very time consuming because I have learned similar functionality during the workshop.

Part C Process

Based on the user testing result, I have added the tooltips for navigation items. Therefore, when the user hovers the navigation button, a message about what the button does will show up.
